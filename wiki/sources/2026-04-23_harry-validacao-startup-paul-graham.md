# 5 Prompts para Validar Sua Ideia de Startup como Paul Graham

> **Fonte:** [[2026-04-23_harry_DXeq-yFkqRP]] | **Autor:** @Harry (@codingknowledge) | **Data:** 2026-04-23 | **Formato:** carousel | **[↗ Ver post](https://www.instagram.com/p/DXeq-yFkqRP/?img_index=5&igsh=b2djMHh4dzFkbGl2)**

## TL;DR

5 prompts estruturados em ROLE/TASK/STEPS/RULES/OUTPUT que aplicam o framework de avaliação de startups de [[paul-graham]] — desde testar a suposição central até lançar um MVP em 2 semanas.

## Contexto

A maioria das ideias de startup fracassa não por ser ruim, mas por nunca ter sido testada antes de meses de desenvolvimento. O autor sistematiza o modo como [[paul-graham]] (co-fundador da YC) avalia candidatos — identificando falhas fatais antes de qualquer linha de código.

## O que foi ensinado

Os 5 prompts cobrem as etapas sequenciais de validação:

### Prompt 1 — Pressure Test Your Idea
- **ROLE**: Avaliador de startups no estilo Paul Graham / YC
- **TASK**: Testar brutalmente a ideia, encontrando toda falha fatal antes de desperdiçar um mês
- **OUTPUT**: Suposição Central → Falhas Fatais (por severidade) → Validação do Problema → Founder-Market Fit → Veredicto Brutal
- **Regra-chave**: Veredicto direto — nunca "tem potencial mas"

### Prompt 2 — Validate the Real Problem
- **ROLE**: Especialista em customer discovery aplicando o framework "fale com os usuários" de PG
- **TASK**: Verificar se o problema é real (painkiller) ou inventado (vitamina)
- **OUTPUT**: Dor Específica → Perfil do Early Adopter → Perguntas de Discovery → Flag vitamina/painkiller

### Prompt 3 — Map Real Competitors
- **ROLE**: Analista de inteligência competitiva com framework "o que as pessoas fazem agora"
- **TASK**: Mapear todos os concorrentes, incluindo os invisíveis
- **Regra-chave**: "Não temos concorrentes" é sempre errado — o comportamento atual do cliente é sempre um concorrente
- **OUTPUT**: Comportamento Atual → Concorrentes Diretos → Concorrentes Indiretos → Verdadeiro Inimigo → Diferenciação Genuína

### Prompt 4 — Find First 10 Customers
- **ROLE**: Especialista em tração inicial aplicando "do things that don't scale" de PG
- **TASK**: Plano específico para encontrar e converter os primeiros 10 clientes manualmente
- **Regra-chave**: Sem automação nesta fase — abordagem manual obrigatória; critérios de sucesso devem ser comportamentais
- **OUTPUT**: Onde Estão os 10 → Abordagem Manual → Primeira Mensagem → Critérios de Sucesso → Plano Semanal

### Prompt 5 — Build MVP in 2 Weeks
- **ROLE**: Arquiteto de MVP aplicando o framework de PG
- **TASK**: Menor versão do produto que teste a suposição central, lançada em 2 semanas para usuários reais
- **Regra-chave**: O MVP testa a suposição mais arriscada — tudo que não testa essa suposição é cortado
- **OUTPUT**: Suposição Central → Conjunto Mínimo de Features → O Que É Cortado → Critérios de Teste → Plano de 2 Semanas

## Insights para o wiki

- **Validação como processo pré-build**: a sequência dos 5 prompts forma um funil completo — ideia → problema → mercado → tração → produto. Complementa e antecede os 7 prompts de estratégia de negócios de [[evolving-ai]] / [[business-bulls]] (que começam depois que o produto já existe)
- **ROLE/TASK/STEPS/RULES/OUTPUT**: todos os 5 prompts usam essa estrutura — reforça que este é o padrão consolidado para prompts de alto desempenho no domínio de negócios (já documentado em [[2026-04-22_castilho-6-palavras-claude]], [[god-of-prompt]])
- **Mentalidade "do things that don't scale"**: presente no Prompt 4 — alinha com o padrão anti-automação precoce que aparece em outras fontes do wiki
- **Paul Graham como referência de autoridade**: usar o nome de um investidor conhecido como persona do ROLE aumenta a especificidade da avaliação — técnica análoga ao uso de "Tim Ferriss" em [[god-of-prompt]]
